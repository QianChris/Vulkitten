1. 实现 GLCommandBuffer 的 BindPipeline 方法：通过 OpenGLDevice 的 GpuSlot 池查询 PipelineHandle 对应的 GLuint program，调用 glUseProgram 并差量绑定（m_CurrentPipelineId 去重）。在 OpenGLDevice 头文件中暴露 GetSlot 方法为 public（当前为 private）。
2. 实现 GLCommandBuffer 的 BindGeometry + 惰性 VAO 创建：通过 Handle 查询 GeometryDesc 对应的 BufferHandle 数组，再从 OpenGLDevice slot 池解析出 GLuint VBO/IBO。首次绑定时根据当前绑定 Pipeline 的 VertexLayout（存在 Pipeline slot 中）创建 VAO（glGenVertexArrays + glVertexAttribPointer）。后续绑定直接 glBindVertexArray 缓存的 VAO。
3. 实现 GLCommandBuffer 的 BindTexture 方法：通过 slot 索引 + Handle 查询 OpenGLDevice slot 池获取 GLuint texture 和 GLuint sampler。调用 glActiveTexture(GL_TEXTURE0 + slot) → glBindTexture → glBindSampler。
4. 实现 GLCommandBuffer 的 PushConstants 方法：将 data 指针指向的数据复制到内部 staging buffer，通过 glUniform* 系列函数上传。MVP 矩阵走 glUniformMatrix4fv，其他小数据走 glUniform1f/2f/3f/4f。
5. 实现 GLCommandBuffer 的 Draw 和 DrawIndexed 方法：分别调用 glDrawArrays 和 glDrawElements（需要已绑定的 VAO）。索引类型根据 Geometry 创建时记录的 IndexType（UInt16→GL_UNSIGNED_SHORT，UInt32→GL_UNSIGNED_INT）决定。
6. 实现 OpenGLDevice 的 createShader 方法：通过 ShaderBytecode 中的 SPIR-V 数据（魔数 0x07230203 检测）创建 GL program。复用现有 OpenGLShader 类的 glShaderBinary + glSpecializeShader + glLinkProgram 流程，返回 ShaderHandle。
7. 实现 OpenGLDevice 的 createPipeline 方法：根据 PipelineDesc 中的 VertexShaderHandle + FragmentShaderHandle 从 slot 池解析 GLuint shader module，创建 GL program 并链接（glAttachShader + glLinkProgram）。读取 PipelineDesc::TextureSlots/BufferSlots 构建 uniform location→texture unit 映射表，存入 slot metadata 供 GLCommandBuffer::BindTexture 使用。
8. 实现 OpenGLDevice 的 createGeometry 方法：根据 GeometryDesc 创建 VBO（glGenBuffers + glBufferData）和 IBO（可选），返回 GeometryHandle。将 vertexCount/indexCount/indexType 存入 slot metadata。
9. 实现 OpenGLDevice 的 beginFrame/endFrame 方法（替换遗留 Submit）：beginFrame 返回 FrameContext（含递增 ring buffer 帧索引），endFrame 调用 glfwSwapBuffers。移除遗留的 Submit 方法。
10. 重构 OpenGLRenderer 使用 IDevice::endFrame 替代 Submit：BeginFrame 调用 Renderer::BeginFrame()→IDevice::beginFrame()，EndFrame 调用 Renderer::EndFrame()→IDevice::endFrame(ctx)。移除 m_RendererAPI 成员（OpenGLRendererAPI）及所有引用。更新 OpenGLRenderer::Init 移除 OpenGLRendererAPI 创建。
11. 重构 SpriteRenderPass 使用 ICommandBuffer 绘制：构造函数中通过 IDevice 创建 QuadVB/QuadIB→GeometryHandle、TextureShader→ShaderHandle→PipelineHandle。Flush() 方法中获取 ICommandBuffer（通过 IDevice::createCommandBuffer），录制 BeginRenderPass→BindPipeline→BindGeometry→PushConstants(VP matrix)→BindTexture(slot,tex,sampler)→DrawIndexed→EndRenderPass。移除 m_QuadVA、m_TextureShader、m_QuadVB 的 Ref<> 成员，改用 rhi::Handle。顶点格式定义在 PipelineDesc::vertexLayout。
12. 重构 PreparePass 使用 ICommandBuffer 清屏：execute 回调中获取 ICommandBuffer，调用 BeginRenderPass 传入 clearValues 数组（解析 ClearCommand）。移除对 OpenGLRendererAPI 的直接调用。
13. 删除 EndPass：SwapBuffers 已在 IDevice::endFrame 中处理。更新 OpenGLRenderer::Init 移除 EndPass 注册，RenderGraph 移除对应 Pass。
14. 实现 VulkanDevice 的 createBuffer 和 createTexture 方法：createBuffer→vkCreateBuffer + vkAllocateMemory（VMA 或手动分配）→Handle。createTexture→vkCreateImage + vkAllocateMemory + vkCreateImageView→Handle。使用内部 GpuSlot 池存储 VkBuffer/VkImage/VkImageView。
15. 实现 VulkanDevice 的 createShader 和 createPipeline 方法：createShader→vkCreateShaderModule（SPIR-V bytecode）→Handle。createPipeline→根据 PipelineDesc 构建 VkGraphicsPipelineCreateInfo（ShaderStage、VertexInput、InputAssembly、Viewport/Scissor、Rasterization、Multisample、DepthStencil、ColorBlend、PipelineLayout）→vkCreateGraphicsPipelines→Handle。读取 TextureSlots/BufferSlots 构建 VkDescriptorSetLayout。
16. 实现 VulkanDevice 的 beginFrame/endFrame 方法：beginFrame→vkWaitForFences→vkAcquireNextImageKHR→vkBeginCommandBuffer→返回 FrameContext。endFrame→vkEndCommandBuffer→vkQueueSubmit→vkQueuePresentKHR。使用 VkFrameContext 管理 per-frame command pool/fence/semaphore。
17. 创建 VkCommandBuffer 实现 rhi::ICommandBuffer：Barrier→vkCmdPipelineBarrier、BeginRenderPass→vkCmdBeginRenderPass、BindPipeline→vkCmdBindPipeline、BindGeometry→vkCmdBindVertexBuffers+vkCmdBindIndexBuffer、BindTexture→push descriptor 写入 VkDescriptorImageInfo、PushConstants→vkCmdPushConstants、Draw/DrawIndexed→vkCmdDraw/vkCmdDrawIndexed、DispatchCompute→vkCmdDispatch。所有方法内部持有 VkCommandBuffer handle 和 VulkanDevice 引用。
18. 实现 VulkanDevice 的 createRenderPass/createFramebuffer/onResize 方法：createRenderPass→vkCreateRenderPass（根据 AttachmentDesc 构建）。createFramebuffer→vkCreateFramebuffer（绑定 VkImageView 到 RenderPass）。onResize→vkDeviceWaitIdle + 重建 swapchain + 重建所有 framebuffer。
19. 重构 VkRenderer 使用 IDevice::beginFrame/endFrame 替代 Submit：BeginFrame 委托 m_Device->beginFrame()，创建 ICommandBuffer。Execute 调用 RenderGraph::Execute()（Pass 录制到 VkCommandBuffer）。EndFrame 调用 m_Device->endFrame(ctx)。移除遗留 Submit 调用。
20. 清理 Vulkan 旧 stub：删除 VkRenderContext.h/.cpp（已被 VkCommandBuffer 替代）、VkShader.h/.cpp（已被 VulkanDevice::createShader 替代）、VkPipeline.h/.cpp（已被 VulkanDevice::createPipeline 替代）。保留 VkFrameContext 用于 multi-frame-in-flight 管理。
21. 实现 VkGpuResourceManager 对接新 Handle 系统：存储 Handle→Vulkan 资源映射（VkBuffer/VkImage/VkImageView/VkSampler 等），保留 TickFrame/Gc（fence-based deferred deletion 替代帧计数 GC）。
